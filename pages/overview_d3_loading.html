Loading CSV Data through D3

Now, it is time for us to take a look at a basic usage of D3, loading data. This is a simple thing to do, but will lay the ground work for the more difficult part of D3.

This overview will include:

  1. Index Setup
  2. Data Setup
  3. Overview of Functions Used

Index Setup

Once the permissions have been set, we need to create a file in which the webserver will serve up. But to make things easy, we will directly copy template into the file. 


      cd ~/public_html
      wget http://cis.gvsu.edu/~oboylela/D3-Lab/examples/d3_load.html
      mv d3_load.html index.html
      chmod 755 index.html

Data Setup

The data required by d3_load.html is currently hosted at here. To properly use this data with d3_load.html, we will need to make a slight modification of d3_laod.html. To do this, open d3_load.html with a text editor, and go to line 14: 


      var filename = "../data/creditCard-mod.csv";

And modify this line to:


      var filename = "creditCard-mod.csv";

Or to whatever you decided to name it, but make sure it is located in ~/public_html. Similiar to what we did with the indxe.html, we need to modify the permissions so that we can access it from our page. To do this, execute these commands: 


      cd ~/public_html
      chmod 744 creditCard-mod.csv

The last part of the chmod 744 creditCard-mod.csv needs to be whatever the filename you chose when you downloaded the file. Now, if you followed the steps above properly, then going to cis.gvsu.edu/~USERNAME should look like this. If it does not work, then verify the permissions are correct, that the files exist where you say it is, and that everything is correctly named.
