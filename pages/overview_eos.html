Setting up the site

To correctly use the D3, there needs to be a webserver to process the results, and as apart of the CIS program, you have access to an account that can be used for this, but we need to do some minimal setup for it to work.

Thi setup will include:

  1. Logging In
    1. X2Go
    2. SSH
  2. Permissions

Logging In!

By now, everyone should have received and email that gave them a username and password. If not, please contact the current Systems Administrator to help you out.

Now, to use X2Go or SSH. X2Go will give you a full virtual desktop that will make it feel as if you were sitting at the computer itself, with graphical interface. SSh will give you only a temrinal line to use, meaining no mouse.

Note! If using X2go, make sure the system used has the required version 4.0.5.0

Permissions

Once logged into the systems, the first thing that needs to be done is to allow the webserver to read the public_html directory. The bash commands below need to be run on the terminal. 


      chmod o+x ~
      chmod o+x ~/public_html

Index Setup

Once the permissions have been set, we need to create a file in which the webserver will serve up. But to make things easy, we will directly copy template into the file. 


      cd ~/public_html
      wget http://cis.gvsu.edu/~oboylela/D3-Lab/examples/d3_load.html
      mv d3_load.html index.html
      chmod 755 index.html
